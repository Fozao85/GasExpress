# Sprint 03 — Ordering & Checkout

## Objective
Enable customers to select LPG products, manage their cart, complete checkout, process payment, and create delivery-ready orders that are sent to vendors.

## Backend
- Cart APIs (with validation)
- Address management
- Order APIs
- Checkout / order processing
- Payment APIs (initiate, verify, callback)
- Vendor notification on order creation

## Frontend
- Cart
- Address selection
- Payment method
- Checkout summary
- Order confirmation

## Deliverables
Customers can complete a full purchase flow — from cart to confirmed order with vendor notification.

## Definition of Done
- Cart functionality works end-to-end with validation
- Checkout flow complete (address → payment → confirmation)
- Order created in database with status tracking
- Stock reserved upon successful order
- Vendor notified after order placement
- Payment processed and verified
- Confirmation screen displayed
- Empty / error / loading states handled
