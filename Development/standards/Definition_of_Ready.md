# Definition of Ready

A user story or task is considered **Ready** for implementation when all of the following criteria are met:

## Requirements
- [ ] User story clearly defined with acceptance criteria
- [ ] Business value and context understood
- [ ] Edge cases and failure scenarios identified
- [ ] Dependencies identified and resolved or noted

## Technical
- [ ] Sprint document updated with Backend_Task / Frontend_Task breakdown
- [ ] API contracts defined (request/response format, status codes)
- [ ] Database schema changes designed and reviewed
- [ ] UI mockups or design references available
- [ ] Auth/authorization requirements specified

## Planning
- [ ] Effort estimated (story points or time)
- [ ] Task broken into implementable units (< 1 day per unit)
- [ ] Team capacity confirmed for the sprint
- [ ] Acceptance criteria clear enough to test against

## Dependencies
- [ ] No unfulfilled sprint dependencies blocking this task
- [ ] External service access confirmed (API keys, sandbox accounts)
- [ ] Test data or fixtures prepared (where needed)

## AI Readiness (for AI-assisted development)
- [ ] AI prompt file (`AI_Prompt.md`) written for the sprint
- [ ] Relevant sprint documents referenced in the prompt
- [ ] Standards documents referenced: coding, API, git workflow
- [ ] Expected output format specified (code, tests, docs)
